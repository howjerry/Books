## 🎯 目錄生成方法論

### 數據提取流程

1. **文件發現**
   - 使用 `Glob` 工具定位所有 `.md` 文件
   - 確認文件存在性和可讀性

2. **標題提取**
   - 使用 `Grep` 工具提取所有 Markdown 標題 (`^#{1,4} `)
   - 針對每個章節文件獨立提取

3. **結構化處理**
   - 解析標題層級（#, ##, ###, ####）
   - 維護原始章節順序
   - 生成 GitHub 風格的錨點鏈接

4. **統計計算**
   - 計算行數、代碼塊數、標題數
   - 估算閱讀時間
   - 生成統計報表

### 錨點鏈接生成規則

GitHub Markdown 錨點鏈接規則:
1. 轉換為小寫（中文不變）
2. 空格替換為連字符 `-`
3. 移除特殊字符（保留中文、英文、數字、連字符）
4. 多個連續連字符合併為一個

**範例**:
- `第 1 章：AI 自動化入門` → `#第-1-章ai-自動化入門`
- `1.2.3 Skills vs. Tools` → `#123-skills-vs-tools`
