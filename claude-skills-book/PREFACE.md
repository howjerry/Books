# Preface - 前言

## 為什麼寫這本書

2024 年，我第一次看到 Stagehand 用自然語言執行瀏覽器測試時，震撼無比。沒有脆弱的 CSS 選擇器，沒有複雜的等待邏輯，只需要告訴 AI：「登入系統」、「加入購物車」、「完成結帳」。測試代碼從數百行縮減到十幾行，維護時間從每週幾小時降到幾乎為零。

這不僅是效率的提升——這是測試自動化的範式轉移。

當 Anthropic 推出 Claude Code Skills 框架時，我意識到這個轉變遠比單純的瀏覽器自動化更深遠。Skills 讓我們能夠教導 AI 執行任何自動化任務：API 測試、數據處理、報告生成、系統監控。AI 不再只是輔助工具，而是成為了自動化系統的核心引擎。

但當我開始探索這個新領域時，發現了一個問題：缺乏系統性的學習資源。網上的文章零散、淺嘗輒止；官方文檔詳盡但缺少實戰指引。市面上沒有一本書能從基礎到生產級系統，完整講解如何使用 Claude Skills 建構真實應用。

於是，我決定寫這本書。

## 這本書的目標

這不是一本理論書籍，也不是 API 文檔的重複。這是一本**實戰指南**，帶你從零開始，一步步建構一個生產就緒的測試自動化系統——**WebGuard**。

通過建立 WebGuard，你將學習：

- **Skills 核心架構**：從 SKILL.md 語法到三層漸進式揭露系統
- **Stagehand 瀏覽器自動化**：AI 驅動的自愈測試
- **API 與數據處理**：完整的測試覆蓋策略
- **Skills 編排與組合**：構建複雜的工作流程
- **CI/CD 整合**：將 Skills 融入開發流程
- **生產級架構**：四層架構、容錯、監控、擴展
- **企業部署**：Kubernetes、安全、MCP 生態

更重要的是，你將學會**如何思考 AI 驅動的自動化**——不是用 AI 取代現有工具，而是重新設計自動化系統的方式。
