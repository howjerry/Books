# 前言

## 為何寫這本書

這不是一本講解 RAG 原理的理論書，而是一本教你動手打造生產級 RAG 系統的實戰指南。

在過去兩年，我見過太多團隊在 RAG 專案上跌倒——不是因為他們不懂技術，而是因為他們低估了「搜尋不準」這個問題的複雜性。向量資料庫裝好了，Embedding 模型跑起來了，LLM 也接上了，但使用者還是抱怨：「為什麼問一個簡單問題，AI 都答不對？」

這本書正是為了解決這個問題而寫。

## 本書的獨特之處

### 1. 專案驅動，而非概念驅動

從第一章開始，你就會動手實作一個真實的專案——「AskBot」企業客服知識庫 AI 助理。這個專案會陪伴你走完全書 13 章，每一章都會為它增加新的能力：

- 第 1-3 章：從關鍵字搜尋演進到基礎 RAG
- 第 4-7 章：透過 Chunking、Re-Ranking、Hybrid Search 大幅提升精準度
- 第 8-10 章：部署到生產環境，建立監控體系
- 第 11-13 章：整合使用者回饋，讓系統自動進化

### 2. 程式碼優先

每一章都會產出可直接執行的程式碼。我不會給你虛擬碼或概念圖——你會得到完整的 Python 模組、Docker Compose 設定檔、Kubernetes 部署清單、Grafana 儀表板設定。

### 3. 量化一切

「這個改動讓系統變好了」是不夠的。你需要知道：

- Precision@5 從 0.65 提升到 0.85
- 延遲 P95 從 300ms 降到 150ms
- 幻覺率從 15% 下降到 3%

本書會教你建立完整的評估框架，讓每一次優化都有數據支撐。

## 你會學到什麼

完成本書後，你將能夠：

1. **從零構建生產級 RAG 系統**：從文件切割、Embedding 選型，到檢索優化與答案生成，掌握每個環節的最佳實踐

2. **量化評估與優化精準度**：建立標準化的測試集與評估框架，科學地比較不同設計方案

3. **部署與監控**：將 RAG 系統部署到 Kubernetes，建立即時監控與告警機制

4. **持續學習與自動化**：整合使用者回饋，建立自動重訓練與灰度發布流程

## 閱讀本書的準備

### 技術背景

- **必備**：Python 程式設計經驗（2 年以上）
- **有幫助**：基本的機器學習概念（不需要深入數學）
- **有幫助**：Docker 與 API 開發經驗

### 開發環境

- Python 3.9+
- Docker Desktop
- 至少 8GB RAM（執行向量資料庫）
- Claude API 金鑰（用於 LLM 呼叫）

### 學習建議

1. **動手實作**：不要只是閱讀程式碼，一定要親手執行
2. **從頭到尾**：章節之間有連貫性，跳過前面的章節可能會讓後面的內容難以理解
3. **使用你自己的資料**：書中會提供範例資料集，但最好的學習方式是用你熟悉的資料來實驗

## 本書的侷限

我不會假裝 RAG 是萬能的。在第 13 章的批判與挑戰章節，我會誠實地討論：

- 量化指標的盲點
- 持續學習的資源與風險成本
- 「越用越準」承諾的現實侷限

這些討論不是為了潑冷水，而是為了幫助你在實際專案中做出更好的決策。
