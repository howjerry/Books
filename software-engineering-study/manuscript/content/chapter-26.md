# 第26章　風險管理

> 「風險管理不是消除風險，而是有準備地面對風險。」

---

## 26.1 什麼是風險？

### 26.1.1 風險的定義

**風險（Risk）**：可能發生並對專案產生負面影響的不確定事件。

| 要素 | 說明 |
|------|------|
| **不確定性** | 可能發生也可能不發生 |
| **影響** | 如果發生會有負面影響 |
| **可預見** | 可以識別和評估 |

### 26.1.2 風險 vs. 問題

| 風險（Risk） | 問題（Issue） |
|--------------|---------------|
| 未來可能發生 | 已經發生 |
| 預防和準備 | 解決和處理 |
| 降低發生機率 | 減少影響 |

### 26.1.3 軟體專案常見風險

| 類別 | 風險範例 |
|------|----------|
| **需求** | 需求不清、需求變更 |
| **技術** | 新技術、整合問題 |
| **人員** | 關鍵人員離職、技能不足 |
| **時程** | 估算不準、相依延遲 |
| **外部** | 供應商問題、法規變更 |


## 26.3 風險識別

### 26.3.1 識別方法

| 方法 | 說明 |
|------|------|
| **腦力激盪** | 團隊一起想可能的風險 |
| **檢查清單** | 使用歷史風險清單 |
| **專家訪談** | 詢問有經驗的人 |
| **SWOT 分析** | 分析弱點和威脅 |
| **假設分析** | 檢視假設是否正確 |

### 26.3.2 風險識別問題

| 問題 | 找什麼風險 |
|------|-----------|
| 什麼可能出錯？ | 技術、需求風險 |
| 我們假設了什麼？ | 假設風險 |
| 我們依賴誰？ | 相依風險 |
| 什麼是新的？ | 技術風險 |
| 什麼是複雜的？ | 實作風險 |
| 什麼是關鍵的？ | 關鍵路徑風險 |

### 26.3.3 風險清單範例

| ID | 風險描述 | 類別 | 來源 |
|----|----------|------|------|
| R1 | 關鍵開發者離職 | 人員 | 專案經理 |
| R2 | 第三方 API 不穩定 | 技術 | 開發團隊 |
| R3 | 需求在開發中變更 | 需求 | 歷史經驗 |
| R4 | 效能不符合要求 | 技術 | 架構師 |
| R5 | 整合測試延遲 | 時程 | 測試經理 |

## 26.4 風險分析

### 26.4.1 風險評估

評估兩個維度：
- **機率（Probability）**：發生的可能性
- **影響（Impact）**：發生後的嚴重程度

### 26.4.2 風險矩陣

```
┌──────────────────────────────────────────────────────────────┐
│                      風險矩陣                                 │
│                                                              │
│         │        影  響                                      │
│         │   低      中      高                               │
│    ─────┼───────────────────────                            │
│      高 │  中等    高      極高                              │
│  機     │                                                    │
│  率  中 │  低      中等    高                                │
│         │                                                    │
│      低 │  低      低      中等                              │
│    ─────┴───────────────────────                            │
│                                                              │
│    風險等級：                                                │
│    極高 ■ 立即處理                                           │
│    高   ■ 優先處理                                           │
│    中   ■ 監控處理                                           │
│    低   □ 接受                                               │
└──────────────────────────────────────────────────────────────┘
```

### 26.4.3 風險評分

| 維度 | 等級 | 說明 | 分數 |
|------|------|------|------|
| **機率** | 高 | > 50% | 3 |
|  | 中 | 20-50% | 2 |
|  | 低 | < 20% | 1 |
| **影響** | 高 | 專案失敗 | 3 |
|  | 中 | 顯著延遲 | 2 |
|  | 低 | 輕微影響 | 1 |

**風險分數** = 機率 × 影響

| 分數 | 等級 | 行動 |
|------|------|------|
| 7-9 | 極高 | 立即處理 |
| 4-6 | 高 | 優先處理 |
| 2-3 | 中 | 監控 |
| 1 | 低 | 接受 |

## 26.5 風險應對

### 26.5.1 應對策略

| 策略 | 說明 | 範例 |
|------|------|------|
| **迴避** | 消除風險來源 | 不使用有風險的技術 |
| **轉移** | 將風險轉給他人 | 外包、保險 |
| **減輕** | 降低機率或影響 | 技術審查、備案 |
| **接受** | 承擔風險 | 準備緩衝、應變計畫 |

### 26.5.2 策略選擇

| 風險等級 | 建議策略 |
|----------|----------|
| 極高 | 迴避或轉移 |
| 高 | 減輕 |
| 中 | 減輕或接受（主動） |
| 低 | 接受（被動） |

### 26.5.3 應對計畫範例

| 風險 | 策略 | 應對行動 | 觸發條件 | 負責人 |
|------|------|----------|----------|--------|
| R1：關鍵開發者離職 | 減輕 | 知識分享、文件化 | - | PM |
|  | 接受 | 準備替補人選 | 離職通知 | HR |
| R2：第三方 API 不穩定 | 減輕 | 加入快取、重試機制 | - | 開發 |
|  | 迴避 | 準備替代 API | 停機 > 1天 | 架構 |
| R3：需求變更 | 減輕 | 變更控制流程 | - | PM |
|  | 接受 | 預留緩衝 | - | PM |

## 26.6 風險監控

### 26.6.1 監控活動

| 活動 | 頻率 | 產出 |
|------|------|------|
| **風險審查會議** | 每週/雙週 | 風險狀態更新 |
| **風險指標追蹤** | 持續 | 預警 |
| **新風險識別** | 持續 | 更新風險清單 |
| **觸發條件監控** | 持續 | 應變啟動 |

### 26.6.2 風險登錄表

| ID | 風險 | 機率 | 影響 | 等級 | 策略 | 狀態 | 負責人 |
|----|------|------|------|------|------|------|--------|
| R1 | 關鍵人員離職 | 中 | 高 | 高 | 減輕 | 監控中 | PM |
| R2 | API 不穩定 | 高 | 中 | 高 | 減輕 | 進行中 | 架構 |
| R3 | 需求變更 | 高 | 中 | 高 | 減輕 | 監控中 | PM |
| R4 | 效能問題 | 低 | 高 | 中 | 接受 | 監控中 | 開發 |

### 26.6.3 風險狀態

| 狀態 | 說明 |
|------|------|
| **識別** | 新識別的風險 |
| **分析中** | 正在評估 |
| **監控中** | 已有應對計畫，監控中 |
| **進行中** | 正在執行應對行動 |
| **已發生** | 風險已轉為問題 |
| **已關閉** | 風險已不存在 |

## 26.7 敏捷中的風險管理

### 26.7.1 敏捷如何降低風險

| 敏捷實踐 | 降低什麼風險 |
|----------|--------------|
| **短迭代** | 早發現問題 |

| **Retrospective** | 持續識別風險 |
| **優先處理高風險** | 早期處理不確定性 |
